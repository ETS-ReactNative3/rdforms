Example 7 - RDForm editor with a registered chooser

In RDForms we can pre-define the available choices to be rendered for a choice item either inline in the template or with a standalone (registered) chooser. We usually prefer the latter when the choices are not static or should be fetched by 3rd parties. In this example we use static data with a standalone chooser to showcase the concept. This chooser can easily modified to fetch dynamic data by modifying chooser.js.

Note how the Publisher has different choices than the Contact Point, Conforms To or Spatial Coverage. The former choices are defined in data.js used by the standalone chooser while the rest uses the default "dummy" RDForm choooser's data. Also note that the chooser can render its own UI (e.g a pop-up or modal) to present and select the choice. The UI would be triggered when the search icon is clicked. Try clicking on the search icon for the Publisher field.
